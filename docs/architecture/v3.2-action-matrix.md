## Severity Legend

- **HIGH**: Implementation contradicts a v3.2 decision or has a correctness bug
- **MEDIUM**: Missing a pattern v3.2 identifies as dominant/universal
- **LOW**: Gap that matters but can wait; stub acknowledged as future work

## HIGH — Fix These (correctness or safety implications)

### 1. D5 Exit priority ordering is inverted
- **Crate:** `neuron-op-react`
- **File:** `op/neuron-op-react/src/lib.rs` ~L988-1052
- **Problem:** Checks max_turns → budget → timeout → hook (ExitCheck). v3.2 mandates: safety halt (hook) > budget > max turns > model done. A guardrail hook that should block execution runs AFTER budget and turn checks.
- **Fix:** Move ExitCheck hook dispatch BEFORE the limit checks. Safety trumps everything.
- **Blocked by:** Nothing. Do first.

### 2. Hook composition short-circuits transformers
- **Crate:** `neuron-hooks`
- **File:** `hooks/neuron-hooks/src/lib.rs` L35-60
- **Problem:** All hook actions short-circuit the pipeline. Two `ModifyToolOutput` hooks (e.g., RedactionHook + formatting hook) — only the first runs. v3.2: transformers should chain (each feeds the next).
- **Fix:** Add `HookKind` enum (Guardrail/Transformer/Observer) to registration. Dispatch by kind: observers first (all run, actions ignored), transformers (chain modifications), guardrails (short-circuit on Halt/Skip).
- **Blocked by:** Nothing. Independent of #1.

### 3. D3C Retry: `ProviderError::RequestFailed` is over-broad
- **Crate:** `neuron-turn`
- **File:** `turn/neuron-turn/src/provider.rs` ~L37-45
- **Problem:** `is_retryable()` marks `RequestFailed` as retryable. But `RequestFailed` includes safety refusals, content filter hits, and auth failures — all non-retryable per v3.2. Only transient network/rate-limit errors should retry.
- **Fix:** Split `RequestFailed` into `TransientError` (retryable) and `ContentBlocked`/`AuthFailed` (not retryable). Providers must classify at the HTTP response level.
- **Blocked by:** Nothing. Provider changes are parallel-safe.

## MEDIUM — Address These (missing dominant patterns)

### 4. D2E Compaction reserve is not enforced
- **Crate:** `neuron-op-react`
- **File:** `op/neuron-op-react/src/lib.rs` ~L1065-1069
- **Problem:** No minimum reserve buffer guaranteed. v3.2 says compaction reserve must never be zero — a system at 100% capacity can't run compaction.
- **Fix:** Add `compaction_reserve_pct` to `ReactConfig` (default 10%). Trigger compaction when used > (1 - reserve) * limit, not at limit.
- **Depends on:** Nothing, but pairs well with #1.

### 5. D2C Memory tiering not exposed
- **Crates:** `neuron-state-memory`, `neuron-state-fs`
- **Problem:** Flat key-value stores. No hot/warm/cold tier distinction. v3.2 says tiers determine what's loaded per-turn vs on-demand vs cross-session search.
- **Fix:** Add optional `MemoryTier` metadata to StateStore reads/writes. Implementations can use it for caching/loading strategy. This is a Layer 0 protocol addition (additive).
- **Depends on:** Layer 0 StateStore trait change.

### 6. D3A No task-type routing
- **Crates:** All providers, `neuron-op-react`
- **Problem:** Model is hardcoded per provider instance. v3.2 says task-type routing (route by complexity) is the dominant production pattern.
- **Fix:** This is a design decision. Options: (a) model selector callback on ReactConfig, (b) ProviderRequest carries routing hints that a smart provider resolves, (c) separate ModelRouter trait. Option (a) is simplest and matches Neuron's opt-in philosophy.
- **Depends on:** Nothing, but lower priority than safety fixes.

### 7. L4 Budget spectrum incomplete
- **Crate:** `neuron-op-react`
- **File:** `op/neuron-op-react/src/lib.rs` ~L60-67
- **Problem:** Only tracks max_turns, max_cost, max_duration. Missing: step limits (per-tool-call count), loop limits (repeated identical calls), per-tool budgets.
- **Fix:** Add to `ReactConfig`. Step limit is cheap (counter). Loop limit needs the stuck-detection from gap analysis Phase 2A.
- **Depends on:** Gap analysis Phase 2A (StuckDetected).

### 8. ExfilGuardHook only checks shell commands
- **Crate:** `neuron-hook-security`
- **File:** `hooks/neuron-hook-security/src/lib.rs` ~L117-145
- **Problem:** Only detects curl/wget patterns. MCP tools, API tools, HTTP tools can all exfiltrate. v3.2 D2D says tools include MCP tools.
- **Fix:** Generalize: check for URL patterns + sensitive data in ANY tool input, not just shell-specific syntax. Add MCP-aware patterns.
- **Depends on:** Nothing.

### 9. Hook errors silently swallowed
- **Crate:** `neuron-hooks`
- **File:** `hooks/neuron-hooks/src/lib.rs` L49-53
- **Problem:** `Err(_e) => continue` with a comment saying "In a real system, this would go to tracing/logging." It's a real system now.
- **Fix:** Accept a `tracing::Span` or callback in HookRegistry. At minimum, `tracing::warn!` on hook error.
- **Depends on:** Adding tracing dependency to neuron-hooks.

## LOW — Track for Later (acknowledged gaps, stubs, future work)

### 10. L1 PreMemoryWrite hook point missing
- Already identified in prior analysis. Add during the effect interpreter work (completed).

### 11. C1/C2 Pre/PostComposition hook points missing
- Composition layer is stubs. Add when building real orchestration.

### 12. D4A Isolation spectrum not consumed by LocalEnv
- Layer 0 defines IsolationBoundary enum. LocalEnv ignores it. Expected — it's "local" env. Add when building container/VM env implementations.

### 13. D4B Vault-mediated credentials stubbed
- neuron-secret-vault is explicitly a stub. Implement when real vault backend needed.

### 14. MCP missing resource/prompt protocol extensions
- neuron-mcp covers tool ops. Resource and prompt extensions are part of MCP spec but lower priority.

### 15. BudgetEvent missing step/loop/wall-clock granularity (Layer 0)
- Lifecycle events don't emit for all L4 budget dimensions. Add when implementing #7.

### 16. CompactionEvent missing outcome/failure path
- Only success path exists. Add error/retry variants when compaction is battle-tested.

### 17. Anthropic provider missing ContentFilter/safety-refusal stop reason mapping
- Maps EndTurn, ToolUse, MaxTokens but not content filter. Fix alongside #3.
